资料来源：NXP AIoT Cloud 软件页面、应用范例页面、系统方案页面和站点地图。部分 API 列表接口需要登录态，未登录时返回 NXP 登录页面，因此当前本地知识库只使用公开可访问页面、静态资源和 Markdown 文档。

软件工具页面公开内容：

| 工具 | 作用 | 适合回答的问题 |
| --- | --- | --- |
| DDR 相关工具体验 | 在不同 DRAM 厂商颗粒上调整 DDR SDRAM 参数，例如频率、驱动强度、ODT，并观察虚拟眼图和功耗变化 | DDR 工具能做什么、为什么要调 DDR 参数、Cloud Lab 如何帮助评估 DRAM 配置 |
| Voice Plugins 相关工具体验 | 利用现有多媒体框架，简化 i.MX 产品上的语音算法集成 | i.MX 上语音算法如何集成、Voice Plugins 的价值是什么 |

应用范例页面公开定位：

- 官网提供不同类型的应用范例，用于帮助开发者理解产品功能。
- 应用范例可作为培训、支持和加快产品上市的资料入口。
- 站点地图显示的范例分类包括 i.MX 产品功能体验、MCU 产品功能体验、合作伙伴产品功能体验。

系统方案页面公开定位：

- 站点地图中包含系统方案入口和 EVSE 开发平台方案详情页。
- 方案页面适合整理为“业务场景 + NXP 硬件 + 软件工具 + 开发流程”的结构化知识。
- 对客服系统而言，方案资料能回答“某个行业方案如何落地”“使用哪些开发板和工具”等问题。

资料边界：

- 公开站点地图可以确认大量页面存在，包括硬件产品页、开发板详情页、应用范例、软件工具和文章。
- 未登录状态下，`get_user_case` 等动态列表接口返回登录页，当前不把需要登录的数据写入知识库。
- 后续如果用户提供登录态或允许人工导出资料，可以继续把更多应用范例、开发板详情和系统方案文档加入 `data/knowledge_base` 并重建索引。

检索关键词：DDR 配置，ODT，驱动强度，虚拟眼图，功耗监测，Voice Plugins，i.MX 语音算法，应用范例，系统方案，EVSE，登录态，资料边界。
